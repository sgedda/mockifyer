📊 Dashboard & Administration

🎛️ Mock Data Dashboard

  • Standalone Dashboard CLI Tool Planned
    Delivery Method: Standalone CLI tool via npx mockifyer-dashboard

    Why Standalone npx?
    • Zero installation: Works immediately with npx - no need to install anything globally or locally
    • No package bloat: Keeps core package lightweight - users who don't need dashboard don't pay the cost
    • Universal access: Works with any project type (Express, Next.js, NestJS, etc.) - not limited to Express users
    • Easy discovery: Simple command to remember: npx mockifyer-dashboard
    • Path detection: Automatically detects mock data path by checking environment variables (MOCKIFYER_PATH), walking up directory tree, or using CLI argument
    • CI/CD friendly: Can be run in pipelines without modifying project dependencies
    • Provider agnostic: Supports both filesystem (JSON files) and SQLite database providers
    • Independent versioning: Dashboard can be updated without affecting core package

    Features:
    • Browse all mock files with search and filtering
    • View request/response data with syntax highlighting
    • Edit mock responses inline with JSON validation
    • Delete and duplicate mocks
    • Visualize request flows and data relationships
    • Statistics dashboard (most used endpoints, mock coverage, etc.)
    • Export/import mock collections
    • Real-time updates when mock files change
    • Support for both filesystem and SQLite providers
    • Auto-detection of mock data path from current directory or environment

    Usage Examples:
    npx mockifyer-dashboard - Auto-detect path
    npx mockifyer-dashboard --path ./mock-data - Explicit path
    MOCKIFYER_PATH=./test-data npx mockifyer-dashboard - Via env var
    npx mockifyer-dashboard --path ./mock-data.db --provider sqlite - SQLite database

    Technical Implementation:
    • Uses existing DatabaseProvider interface for provider abstraction
    • Reuses mock data structure and types from core package
    • Lightweight Express server for API endpoints
    • Modern vanilla JS UI (or minimal framework) for frontend
    • Smart path detection: checks MOCKIFYER_PATH env var, walks up directory tree looking for mock-data folders, supports SQLite files
    • Reads from process.cwd() - works from any directory in the project
    dashboard ui administration cli npx standalone
  • Express Middleware Option (Future) Backlog
    Optional Express middleware for teams that want to mount the dashboard directly in their Express app. Provides tighter integration with existing auth and middleware, but limited to Express projects. Can share core dashboard logic with standalone CLI.

📝 Notes & Thoughts

  • Session Continuity: Consider adding a `continueSession` flag to allow sessions to persist across restarts. Useful for development workflows where you want to track requests across multiple server restarts.
  • Performance: For large sessions with many requests, consider pagination or lazy loading in the visualization. Also think about data retention policies - should old sessions be archived?
  • Privacy: Call stacks might contain sensitive information. Consider options to sanitize or anonymize stack traces before storing.
