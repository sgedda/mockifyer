🎭 Mockifyer

A powerful Node.js library for mocking and recording API calls, with advanced date manipulation for deterministic testing

What is Mockifyer?

Mockifyer is a comprehensive API mocking and testing library designed to make your development and testing workflows more efficient and reliable. It intercepts HTTP requests made by your application and can either replay previously recorded responses or record new ones for future use.

Key Benefits:

  • 🚫 No more API rate limits: Test without worrying about external API quotas
  • ⚡ Faster tests: Mock responses are instant, no network latency
  • 🔒 Deterministic testing: Same inputs always produce the same outputs
  • 📅 Date manipulation: Test time-dependent features with fixed or offset dates
  • 🔄 Easy recording: Automatically capture real API responses for later use
  • 🌐 Works with any HTTP client: Supports Axios, Fetch, and more
  • 🔍 Easy data discovery: All captured API responses are stored as searchable JSON files in your project, making it easy to discover what data was captured, where it comes from, and explore all available mock data directly in your IDE

How to Use Mockifyer in Your Codebase

1. Installation

npm install @sgedda/mockifyer

2. Basic Setup

Initialize Mockifyer at the very beginning of your application, before any HTTP client imports:

import { setupMockifyer } from '@sgedda/mockifyer'; // Initialize before any other imports that use HTTP clients setupMockifyer({ mockDataPath: './mock-data', recordMode: process.env.MOCKIFYER_RECORD === 'true', failOnMissingMock: false }); // Now import your HTTP client (Axios, etc.) import axios from 'axios';

3. Environment Configuration

Control Mockifyer behavior through environment variables:

# Enable Mockifyer MOCKIFYER_ENABLED=true # Path to store/load mock files MOCKIFYER_PATH=./mock-data # Enable recording mode (records real API responses) MOCKIFYER_RECORD=true # Use similar path matching when exact match fails MOCKIFYER_USE_SIMILAR_MATCH=true

4. Using in Your Services

Once initialized, your existing HTTP calls will automatically be intercepted:

// Your code doesn't need to change! const response = await axios.get('https://api.example.com/data', { params: { id: 123 } }); // If MOCKIFYER_RECORD=true, this will save the response // If MOCKIFYER_RECORD=false, this will use a saved mock (if available) // If no mock exists and not recording, it makes the real API call

Key Capabilities

🎯 Request Matching

Mockifyer intelligently matches requests to mock files based on:

  • Method: GET, POST, PUT, DELETE, etc.
  • URL: Full URL including protocol and domain
  • Query Parameters: All query string parameters
  • Request Body: POST/PUT body content
  • GraphQL Queries: Special handling for GraphQL with variable matching

📝 Recording Mode

When enabled, Mockifyer automatically:

  • Makes real API calls and captures responses
  • Saves responses as JSON files with descriptive names (timestamp, method, and endpoint URL)
  • Anonymizes sensitive data (API keys, tokens) by default
  • Organizes files by timestamp and endpoint
  • Creates searchable JSON files in your project, making it easy to discover all captured data, see where it comes from, and explore available mock data directly in your IDE

🔄 Mocking Mode

When recording is disabled, Mockifyer:

  • Searches for matching mock files
  • Returns saved responses instantly (no network calls)
  • Falls back to real API if no mock is found (configurable)
  • Supports similar path matching for flexible testing

📅 Date Manipulation

Test time-dependent features with deterministic dates:

import { getCurrentDate } from '@sgedda/mockifyer'; // Get the manipulated date (fixed, offset, or system date) // IMPORTANT: Use getCurrentDate() instead of new Date() for date manipulation to work const currentDate = getCurrentDate(); // Use in your tests const subscription = { startDate: currentDate, endDate: new Date(currentDate.getTime() + 30 * 24 * 60 * 60 * 1000) };

🔑 Important: For date manipulation to work, you must use Mockifyer's getCurrentDate() function instead of new Date(). The getCurrentDate() function respects your date manipulation settings (fixed date, offset, timezone), while new Date() always returns the system date.

Configure dates via:

  • Fixed Date: Always return a specific date
  • Offset: Add/subtract time from current date
  • Timezone: Handle dates in specific timezones
  • Environment Variables: MOCKIFYER_DATE, MOCKIFYER_DATE_OFFSET, MOCKIFYER_TIMEZONE

🔍 Advanced Features

Similar Path Matching

Match requests even when URLs differ slightly (e.g., different IDs but same endpoint structure)

Response Verification

Verify that similar matches produce compatible responses before using them

GraphQL Support

Special handling for GraphQL queries with variable matching and query normalization

Auto Mock Mode

Fail fast if no mock is found - perfect for strict testing environments

Who is Mockifyer For?

Mockifyer is designed for teams needing deterministic API responses for reliable, predictable development and testing workflows.

🎯 Perfect For:

🧪 Testing Teams

Create realistic unit and integration tests with real API response data. No more flaky tests due to external API inconsistencies or rate limits.

🔍 Data Discovery Teams

Discover and explore all data captured from APIs. All mock files are searchable JSON files in your project, making it easy to find what data is available, where it comes from, and explore it directly in your IDE.

📱 UI/UX Teams

Run your app with predefined data to see how it looks for different predictable scenarios. Test time-dependent UI states like "1 hour before departure" or "payment pending for 3 days" with consistent, reproducible data.

🚀 Development Teams

Work offline, develop features independently, and test edge cases without depending on external APIs or worrying about rate limits.

Key Benefits for Your Team

  • Deterministic Testing: Same inputs always produce the same outputs - no surprises, no flaky tests
  • Realistic Test Data: Use actual API responses captured from real endpoints, not synthetic mock data
  • Searchable Data: All captured API responses are stored as JSON files in your project, fully searchable in your IDE
  • Predictable Scenarios: Test specific time-based or state-based scenarios with predefined data (e.g., "show how the app looks 1 hour before departure", "display payment pending state")
  • Data Discovery: Easily discover what data was captured, where it comes from, and explore all available mock data
  • No External Dependencies: Run tests and demos without internet connection or external API access
