⚙️ Settings

All Configuration Options

Complete reference of all configuration options available in Mockifyer. Use these options when calling setupMockifyer() or as environment variables.

Basic Configuration

Option Type Description Env Var
mockDataPath string Path where mock data files are stored and loaded from MOCKIFYER_PATH
recordMode boolean When true, records real API responses to mock files. When false, uses existing mocks MOCKIFYER_RECORD_MODE
failOnMissingMock boolean When true, throws error if no mock found. Automatically false when recordMode is true MOCKIFYER_AUTO_MOCK

Recording Options

Option Type Description Env Var
recordSameEndpoints boolean When true, allows recording multiple responses for the same endpoint MOCKIFYER_RECORD_SAME_ENDPOINTS
anonymizeHeaders string[] Headers to anonymize when saving mocks (defaults to common API key headers). Empty array to disable -
anonymizeQueryParams string[] Query params to anonymize when saving mocks (defaults to common API key params). Empty array to disable -

Matching Options

Option Type Description Env Var
useSimilarMatch boolean When true, tries to find similar path matches when exact match fails MOCKIFYER_USE_SIMILAR_MATCH
useSimilarMatchCheckResponse boolean When true, verifies response content matches before using similar path match MOCKIFYER_USE_SIMILAR_MATCH_CHECK_RESPONSE
similarMatchRequiredParams string[] Query parameters that must match for similar match (e.g., ['season', 'league']) -
requestMatching object Configure request matching: { headers: string[], ignoreQueryParams: string[] } -

Date Manipulation

Option Type Description Env Var
dateManipulation.fixedDate string | Date Fixed date to use instead of current date (ISO format) MOCKIFYER_DATE
dateManipulation.offset number Offset in milliseconds from current date (positive = future, negative = past) MOCKIFYER_DATE_OFFSET
dateManipulation.timezone string Timezone for date operations (e.g., 'America/New_York') MOCKIFYER_TIMEZONE

HTTP Client Options

Option Type Description Env Var
httpClientType 'axios' | 'fetch' Type of HTTP client to use (default: 'axios') -
baseUrl string Base URL for HTTP client -
defaultHeaders Record<string, string> Default headers to include in all requests -
axiosInstance AxiosInstance Custom Axios instance to use -
useGlobalAxios boolean When true, patches global axios instance -

Scenarios

Option Type Description Env Var
scenarios object Scenario configuration: { default: string, [key: string]: string } MOCKIFYER_SCENARIO

💡 Example Configuration

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: false,
  failOnMissingMock: false,
  useSimilarMatch: true,
  useSimilarMatchCheckResponse: true,
  similarMatchRequiredParams: ['season', 'league'],
  dateManipulation: {
    fixedDate: '2024-01-01T00:00:00Z'
  },
  requestMatching: {
    headers: ['authorization'],
    ignoreQueryParams: ['timestamp']
  },
  anonymizeHeaders: ['x-api-key'],
  anonymizeQueryParams: ['api_key']
})
