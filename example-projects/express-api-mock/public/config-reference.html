⚙️ Settings

All Configuration Options

Complete reference of all configuration options available in Mockifyer. Use these options when calling setupMockifyer() or as environment variables.

Basic Configuration

Option Type Description Env Var
mockDataPath string Path where mock data files are stored and loaded from MOCKIFYER_PATH
recordMode boolean When true, records real API responses to mock files. When false, uses existing mocks MOCKIFYER_RECORD_MODE
failOnMissingMock boolean When true, throws error if no mock found. Automatically false when recordMode is true MOCKIFYER_AUTO_MOCK

Recording Options

Option Type Description Env Var
recordSameEndpoints boolean When true, allows recording multiple responses for the same endpoint MOCKIFYER_RECORD_SAME_ENDPOINTS
anonymizeHeaders string[] Headers to anonymize when saving mocks (defaults to common API key headers). Empty array to disable -
anonymizeQueryParams string[] Query params to anonymize when saving mocks (defaults to common API key params). Empty array to disable -

Matching Options

Option Type Description Env Var
useSimilarMatch boolean When true, tries to find similar path matches when exact match fails MOCKIFYER_USE_SIMILAR_MATCH
useSimilarMatchCheckResponse boolean When true, verifies response content matches before using similar path match MOCKIFYER_USE_SIMILAR_MATCH_CHECK_RESPONSE
similarMatchRequiredParams string[] Query parameters that must match for similar match (e.g., ['season', 'league']) -
requestMatching object Configure request matching: { headers: string[], ignoreQueryParams: string[] } -

Date Manipulation

Option Type Description Env Var
dateManipulation.fixedDate string | Date Fixed date to use instead of current date (ISO format) MOCKIFYER_DATE
dateManipulation.offset number Offset in milliseconds from current date (positive = future, negative = past) MOCKIFYER_DATE_OFFSET
dateManipulation.timezone string Timezone for date operations (e.g., 'America/New_York') MOCKIFYER_TIMEZONE

HTTP Client Options

Option Type Description Env Var
httpClientType 'axios' | 'fetch' Type of HTTP client to use (default: 'axios') -
baseUrl string Base URL for HTTP client -
defaultHeaders Record<string, string> Default headers to include in all requests -
axiosInstance AxiosInstance Custom Axios instance to use -
useGlobalAxios boolean When true, patches global axios instance -

Scenarios

Option Type Description Env Var
scenarios object Scenario configuration: { default: string, [key: string]: string } MOCKIFYER_SCENARIO

💡 Realistic Examples

Here are practical examples based on the playground endpoints you can try:

🌤️ Example 1: Recording Weather API Responses

Record real weather API responses for different cities to use in your tests: 

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: true,  // Enable recording
  failOnMissingMock: false,  // Allow real API calls when no mock exists
  anonymizeHeaders: ['x-rapidapi-key'],  // Hide your API key in saved mocks
  anonymizeQueryParams: ['appid']  // Hide API key in query params
})

// Now make API calls - they'll be saved as mocks
await axios.get('https://api.openweathermap.org/data/2.5/weather?q=London&appid=YOUR_KEY')
await axios.get('https://api.openweathermap.org/data/2.5/weather?q=NewYork&appid=YOUR_KEY')

📅 Example 2: Testing Events with Date Manipulation

Test how your events filtering works with different dates without waiting: 

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: false,
  dateManipulation: {
    offset: 665 * 24 * 60 * 60 * 1000  // 665 days in the future
  }
})

// In your code, use getCurrentDate() instead of new Date()
import { getCurrentDate } from '@sgedda/mockifyer';
const currentDate = getCurrentDate();  // Returns date 665 days in future
const filteredEvents = events.filter(e => e.date > currentDate);

⚽ Example 3: Flexible Matching for Football Standings

Use similar match to handle different league IDs while ensuring season matches: 

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: false,
  useSimilarMatch: true,
  useSimilarMatchCheckResponse: true,  // Verify response structure matches
  similarMatchRequiredParams: ['season'],  // Season must match exactly
  requestMatching: {
    ignoreQueryParams: ['timestamp', 'cache']  // Ignore cache-busting params
  }
})

// These will match if season matches, even with different league IDs:
// /standings?league=39&season=2022 → matches mock for league=140&season=2022
// /standings?league=39&season=2023 → won't match (different season)

🎭 Example 4: Testing Different Scenarios

Test different API response scenarios (success, error, empty) easily: 

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: false,
  scenarios: {
    default: 'success',
    'api-error': 'error-response',
    'no-data': 'empty-response'
  }
})

// Switch scenarios via environment variable:
// MOCKIFYER_SCENARIO=api-error  // Use error response mocks
// MOCKIFYER_SCENARIO=no-data    // Use empty response mocks
// MOCKIFYER_SCENARIO=success    // Use normal response mocks

🔒 Example 5: Strict Testing Mode

Fail fast if mocks are missing - perfect for CI/CD pipelines: 

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: false,
  failOnMissingMock: true,  // Throw error if mock not found
  useSimilarMatch: false,    // Require exact matches only
  dateManipulation: {
    fixedDate: '2024-12-25T12:00:00Z'  // Deterministic dates for tests
  }
})

// Perfect for unit/integration tests where you want:
// - Predictable, deterministic responses
// - Failures if mocks are missing (catch missing test data)
// - No accidental real API calls

🔷 Example 6: GraphQL Query Matching

Mockifyer automatically handles GraphQL queries with variables: 

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: true
})

// GraphQL queries are normalized and matched by:
// - Query structure (normalized)
// - Variables (hashed)
// - Endpoint URL

// These will be matched correctly:
// Query with page=1 → saved as mock
// Query with page=3 → uses same mock (if structure matches)
// Different query structure → creates new mock

📋 Complete Configuration Example

Here's a comprehensive example combining multiple options: 

setupMockifyer({
  mockDataPath: './mock-data',
  recordMode: false,
  failOnMissingMock: false,
  useSimilarMatch: true,
  useSimilarMatchCheckResponse: true,
  similarMatchRequiredParams: ['season', 'league'],
  dateManipulation: {
    fixedDate: '2024-01-01T00:00:00Z'
  },
  requestMatching: {
    headers: ['authorization'],
    ignoreQueryParams: ['timestamp']
  },
  anonymizeHeaders: ['x-api-key'],
  anonymizeQueryParams: ['api_key']
})
